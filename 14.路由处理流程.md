# 1.路由映射表

```
在 Django 中，路由映射表是指 URL 配置文件中定义的路由规则和相应的视图函数或类之间的映射关系。它将特定的 URL 请求映射到相应的处理函数，以实现请求的分发和处理。

路由映射表是通过 URL 配置文件来定义的，通常是项目根目录下的 urls.py 文件。在该文件中，您可以使用 path() 或 re_path() 函数来定义路由规则，并将其与对应的视图函数或类关联起来。
```



# 2.路由处理流程

```
客户端发起 HTTP 请求，例如访问网站的某个页面。

Django 的服务器接收到请求，并将其传递给 Django 的核心处理器，即 WSGI（Web 服务器网关接口）或 ASGI（异步 Web 服务器网关接口）。

核心处理器将请求传递给 Django 的中间件（Middleware）。中间件可以执行各种功能，如身份验证、日志记录等。

中间件将请求传递给 Django 的 URL 解析器。

URL 解析器根据项目的 URLconf 配置找到与请求匹配的 URL 模式。

匹配成功后，URL 解析器将请求转发给相应的视图函数或类。

视图函数或类处理请求并生成响应。

响应经过中间件再次处理，然后返回给核心处理器。

核心处理器将响应发送给客户端。
```

```
在 Django 中，URLconf 配置通常位于项目的主 URL 配置文件（urls.py）中。这个文件定义了 URL 模式和对应的视图函数或类。URL 模式可以是简单的字符串，也可以是正则表达式，用于匹配不同的 URL。

URLconf 配置文件中的每个 URL 模式都由一个元组构成，包含了两个元素：URL 正则表达式和视图函数或类。当有请求到达时，Django 会按照配置文件中定义的顺序逐个匹配 URL 模式，直到找到第一个匹配的模式为止。然后，对应的视图函数或类将被调用来处理请求。 ----> 路由映射表
```
